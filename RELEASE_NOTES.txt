Release Notes
Version 3.2.2  (REV1 / v1 hardware ONLY)
July 2026

A REV1 measurement-and-fix release, produced with a logic-analyzer +
debugger closed loop on a real rev 1.0 unit. v2 OWNERS: stay on v3.2 - the
shared-code changes here are validated on v1 hardware only and ship as v3.3
after a v2 regression pass.

Changed (REV1)
--
1. The clock-ratio zones are re-anchored to MEASURED panel positions: the
   Time Multiply pot is audio-taper, so the printed "1" sits at ~49% of the
   electrical range (calibrated ~2013), not at the third the fader-taper
   theory predicted. The old anchor put the printed "1" one count from the
   x2/x3 zone edge - reading x3 at "1" and letting knob jitter flip ratios
   (reported as "pulse width changes with the multiplier knob"). The x1 zone
   is now centred on the measured mark with ~250 counts of margin each side.

Fixed (REV1)
--
1. Short gates were swallowed on externally clocked steps: a step advance
   driven from a pulse interrupt had its first millisecond consumed before
   the outputs were evaluated. Pulse-width setting 0 (the fixed ~1 ms
   trigger) never fired at all while clocked, and every other gate started
   up to 1 ms late. All step entries now latch a pulse onset, so triggers
   fire at full width, aligned to the step.

Bench verification (Saleae, external clock x1, ~850 ms period)
--
- Ratio: 22/22 and 47/47 pulses at exactly 1:1 with the knob on the
  printed "1".
- Pulse width truth curve, all 16 settings: measured within 0.1 percentage
  point of programmed at every setting (PW0 = 1.000 ms trigger, PW15 = 99%).
- Humanize: sliders down = dead on the grid (+0.9 ms +/- 0.2 processing
  latency, zero spread over 47 clocks); sliders at 50% = uniform +/-24% of a
  step; sliders full up = uniform +/-1/2 step (-49.5%..+47.6% measured),
  mean centred on the grid at every depth, lock never lost.
- Lock stability: 10-minute soak, 712/712 pulses, onset +0.98 ms +/- 0.28,
  zero drift between first and last quarter.
- Shift register (Turing) engine, traced at the register level over SWD:
  exactly one shift per clock (24/24), feedback tap correct (24/24), locked
  loop repeats its 8 values exactly; full slip mutates at the statistically
  correct rate and never repeats; parked-stage pulse refire at the clock
  period with the stage's programmed width (53.0% measured).
- Sustain and Enable stage modes under a real gate: with a fast loop and a
  sustain stage, zero stage exits occurred inside the gate-high phase over
  1871 laps (releases align with gate falls); flipped to enable, zero exits
  inside gate-low over 540 laps (releases align with gate rises).

Known issues / notes (3.2.2)
--
- The zone anchors are measured on ONE v1 unit. If v2's Time Multiply pot is
  linear rather than audio-taper, v2 needs its own anchors - measure before
  shipping this in a v2 build.
- Strobe inputs on UNMODIFIED rev 1.0 boards do not work and cannot work:
  the panel jacks are not electrically connected to the MCU (verified with
  a live pulse against every analog channel, the full switch chain, and
  every GPIO). The original rev 1.0 firmware had no strobe support either;
  strobes were added by the v1.6 board+firmware mod (PB2/PB14). The REV1
  firmware's PB2/PB14 strobe path itself is verified working (via injected
  edges): v1.6-modded boards get working strobes, unmodified rev 1.0
  boards do not - this is a hardware limitation, not a firmware bug.


Release Notes
Version 3.2.1  (REV1 / v1 hardware ONLY)
July 2026

A v1-hardware release: the REV1 image is validated on a real v1 unit for the
first time and gains full 3.2 feature parity. v2 OWNERS: stay on v3.2 - this
release intentionally publishes no v2 image (shared improvements that also
touch v2 will ship as v3.3 after a v2 regression pass).

New (REV1)
--
1. Full v3.2 feature parity: external clock sync (ratio knob + humanize
   sliders), the reworked shift-register mode, both-channels programming,
   the gesture fixes and the 50%-gate / 8-bit-register defaults. Previously
   REV1 was functionally frozen at 3.1 behaviour.
2. Shift-register clocks now gate the pulse outputs: when the playing
   stage's register is clocked, the stage's pulses re-fire with the stage's
   pulse-width setting scaled to the incoming clock period.
3. The clock-ratio zones are anchored to the panel legend: the printed "1"
   on Time Multiply = x1 (it is NOT at noon); full CW = x8, full CCW = /8.

Fixed (REV1)
--
1. ALL analog inputs (sliders, Time Multiply, stage-address CVs, external
   inputs) now work: v1 boards need explicit full-chain ADC mux selection
   (ported from the original v1.0 firmware); the optimized v2 addressing
   landed selections wrong, which also explains the historic "Time Multiply
   and continuous stage address dead on v1" reports.
4. v1's Start/Stop inputs are INVERTED by the input conditioning (idle high,
   pulse pulls low). Pulses are now processed once, on the leading edge, so
   external clocking (start+stop) locks correctly; Enable/Sustain stages read
   the true gate state (they were doubly broken before: wrong pin AND wrong
   polarity).
5. The active shift-register clock input is oversampled (~0.3 ms) so musical
   clock pulses register reliably; readings can no longer be attributed to
   the wrong channel (first conversion after each mux move is discarded).
6. An input that is actively clocking a register suppresses the
   step-on-entry soft-normal for 2 s (no more double-clocked, stuttering
   patterns).

Known issues / notes (3.2.1)
--
- No v2 image in this release by design; v2 users stay on v3.2.
- Bench-validated on one v1 unit: analog inputs, external clock sync
  (lock/ratios/humanize), shift-register clocking and gates. NOT yet
  validated on v1: strobe inputs (PB2/PB14), Sustain/Enable under a gate,
  Time/Ref OUT levels (12-bit DAC port), expander units (which also do not
  get the clock-input oversampling).
- With BOTH sequences in Turing mode on different clock inputs, the two
  inputs alternate the fast sampling (each ~0.7 ms) - keep clock pulses
  above ~1.5 ms in that configuration.
- On v2 (v3.2), the shift-register clock inputs still sample at ~3.3 ms:
  use clock pulses/gates of a few ms or slower rates there until v3.3.


Release Notes
Version 3.2
July 2026

New features (v2 hardware only; the v1.6/REV1 build is functionally unchanged)
--

1. External clock sync. Patch a clock into BOTH Start and Stop of a generator
   and it locks to that clock (pulses 20 ms - 2 s apart). While locked:
   - The Time Multiply knob selects an integer clock ratio instead of a
     continuous multiplier: /8 /7 /6 /5 /4 /3 /2 [x1 at the panel's printed
     "1"] x2 ... x8 (full clockwise).
     Multiplied steps are subdivided evenly and re-synced on every incoming
     pulse, so they can never drift. The ratio zones have hysteresis.
   - The time sliders become a per-stage "humanize" amount: slider down = the
     stage lands dead on the clock; raising it adds a random timing shift,
     freshly drawn each time the stage plays - early or late, scaling
     linearly up to half a step at the top - while the long-term grid stays
     locked to the clock. The slider depth is applied live, and gate lengths
     stay steady while onsets move.
   - No clock for 2 seconds -> the generator returns to normal free-running
     (sliders set times, Time Multiply is continuous again).
   - The panel Advance button stays a plain manual advance and never creates
     or disturbs a clock lock.

2. Both-channels programming. Hold BOTH Display switches and the programming
   switches apply to both generators' current stages at once, wherever each
   one is (e.g. flick Quantize to quantize both playing stages). Stage No
   Left/Right shifts both sections while held. Display focus does not change
   during the hold.

3. Shift-register (Turing) mode rework. Each stage is now a classic looping
   shift register: a repeating sequence of `length` (2-16) FULL-RANGE voltages
   played by the stage's assigned clock input (A-D) while the sequencer is on
   that stage. The voltage slider is a monotonic "slip" control: slider down =
   locked (the same voltages repeat in sequence, exactly), slider up = every
   clock slips (the sequence never repeats), with a squared curve in between
   (most of the travel is gentle mutation of the loop). Fixes vs 3.1:
   - values span the full range at any loop length (short lengths used to be
     confined to a sliver of the bottom of the range);
   - the old invert/"double lock" zone is gone; the mapping is monotonic;
   - the slider is read live - its position is always the slip amount, even
     right after loading a program (pinning no longer applies to it);
   - clock inputs trigger at ~2 V (re-arm below ~1 V) so ordinary 5 V clocks
     register reliably (the old mid-scale threshold sat exactly on 5 V);
   - only the currently-playing stage's register is clocked;
   - soft-normalled: with nothing patched into a stage's assigned input, its
     register steps once per stage entry, so the mode works with no cables;
   - new/cleared/randomized/preset stages default to loop length 8.

4. Cleared/fresh programs now default to a ~50% gate on every step. The old
   default was the classic ~1 ms fixed trigger, which is inaudible as a gate
   and made a fresh module feel dead. Pull a step's pulse width all the way
   down (pulse-width chord) to get the classic trigger behaviour back.

5. Gesture fixes.
   - The clock/length config gesture (Source External held) now seeds its LED
     display with the focused stage's actual clock assignment on entry -
     previously it showed a stale value from an earlier gesture until a slider
     crossed the move threshold, which read as "bottom selects B, not A".
   - In the config and scale/root gestures, the selection follows the slider
     being moved NOW. Previously any slider displaced earlier in the hold kept
     re-asserting its value every loop, so a fast throw or a higher-numbered
     slider could mask all later changes until the gesture was re-entered.
   - The Turing-toggle and randomize hold timers tolerate brief (<150 ms)
     dropouts in the switch reading instead of silently restarting.
   - Toggling Turing mode OFF now plays a reverse LED chase. Previously only
     the ON toggle animated, so an off-toggle was silent and easily read as
     "the chord did nothing" - especially since the breathing Source LED only
     shows when the focused stage is set to External.


Release Notes
Version 2.66
June 2022

Functional changes
--
This release brings the v2 marf up to feature parity with Buchla's original unit
and duplicates the 200 series module's behavior to the best of our knowledge.
It includes a large number of functional improvements over the earlier firmwares.

1. Programmed pulses fire correctly in all modes, whenever a step changes.
2. Reference signal present in all modes, whenever a step changes.
3. Steps set to sloped mode will now slew correctly in all modes including
   advance, strobe and continuous address.
4. Quantize mode works correctly on full range steps.
5. Loop behavior is correct. Loops now search backwards from a "last" step to 
   the closest "first" step. Multiple loops are now possible.
6. The "level 2" programming is organized into a single bank.
   It is no longer different between afg1 and afg2.
7. External inputs no longer have audible slewing.
8. Glitches in voltage output fixed.
9. Step times match panel legend exactly.

Internal performance, operational and code health improvements
--
This release required significant refactoring to almost 100% of the codebase,
including all of the core function generation logic, all of the interface code,
and much of the internal scheduling logic. This release is able to run at 8x
the timing resolution of the previous version.

1. Using a much better filtering algorithm consistently on all adc inputs.
   Fixes slewing on large jumps and jitter on steady state signals.
2. Output sample rate increased from 4kHz to 32kHz.
3. Voltage quantization and timing computations use higher precision floating point calculations.
4. Numerous bugs with start/stop and sustain/enable modes ironed out.
5. Slow code moved out of IRQs into main loop and scheduling tightened up.
6. Inlining small methods and compiling with speed optimizations.
7. Reduced function call overhead in peripheral drivers.
8. Improved display update system lights leds better when the clock is at high speeds.
9. Applied consistent code style guide including capitilization, indentation.
10. Improved naming conventions using Buchla standard terminology.
11. Double buffering scheme to reduce glitches at stage transitions


Manual Addendum
--

Calibration Procedure

1. Hold Stage Address 1 Advance down during start up.
2. Leds cycling.
3. Turn all pots to maximum, and apply 10v to all 4 external inputs
4. Select pulse 2 up if you need to swap the pulse leds. Led will move to pulse 2.
5. Select pulse 1 up to swap it back to normal.
6. Press Stage Address 2 Advance down to save calibration.
7. It will ERASE the entire eprom, including saved programs.


Afg Section Shift (unexpanded units only)

1. Afgs 1 and 2 both access the same memory unless you use section shift.
2. Press Stage Address Display 1 + Stage No Right to move afg1 to steps [16-32].
3. Press Stage Address Display 1 + Stage No Left to move afg1 back to steps [1-16].
3. Press Stage Address Display 2 + Stage No Right to move afg2 to steps [16-32].
4. Press Stage Address Display 2 + Stage No Left to move afg2 back to steps [1-16].


Save Program

1. This version supports saving and loading 16 programs (even with expander).
2. Slider data is always saved along with the program.
3. Press the Clear switch _down_ briefly to enter save mode. 
4. The pulse leds will toggle to indicate that you are in save mode.
5. To abort saving, press one of the Display switches.
6. Use the Stage No switch to select one of the 16 memory locations.
7. Press Clear switch _down_ again briefly to save the program.
8. Leds with flash downwards when the program is saved.
9. Note that the marf does not stop running while saving a program.


Load Program

1. Press the Clear switch _up_ briefly to enter load mode.
2. The pulse leds will toggle to indicate that you are in load mode.
3. To abort loading, press one of the Display switches.
4. Use the Stage No switch to select one of the 16 memory locations.
5. Press Clear switch _up_ again briefly to load the program.
6. Leds will flash upwards when the program is loaded.
7. Note that the marf does not stop running when loading a program.
8. Once the program is loaded, all sliders will be "pinned" at their saved value.
9. To restore slider activity, the slider must be moved through the saved value.


Pulse Tricks

1. A simultaneous pulse to start and stop is the same as the manual advance switch.
2. A simultaneous pulse to strobe and start will correctly start on the strobed step.
